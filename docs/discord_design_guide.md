## Design Best Practices

### Category Organization
1. **Information First**: Always put info/rules at the top
2. **Logical Grouping**: Group related channels together
3. **Staff at Bottom**: Keep staff channels at the very bottom
4. **Voice Separate**: Voice channels usually get their own category
5. **Limit Categories**: 5-8 categories is ideal, 10 max

### Channel Naming
1. Use lowercase for text channels
2. Use Title Case for voice channels
3. Keep names short (under 25 characters)
4. Be consistent with emoji placement (all at start OR all at end)
5. Use separators consistently (â”‚, ãƒ», â”ƒ, etc.)

### Permission Structure
1. @everyone should only access public info and chat areas
2. Create role hierarchy: Owner > Admin > Mod > Staff > VIP > Member
3. Use private categories for staff-only sections
4. Read-only for announcements/rules (deny send_messages to @everyone)
5. Separate voice permissions from text if needed

### Visual Consistency
1. **Pick ONE style** and stick with it throughout the server
2. Use the same emoji style throughout
3. Maintain consistent spacing and formatting
4. Don't mix too many font styles (1-2 max)
5. Test readability on both light and dark themes

### Popular Design Tags
High-engagement aesthetic styles:
- `aesthetic` - Clean, visually pleasing
- `cute/kawaii` - Soft, friendly appearance
- `cozy` - Warm, welcoming feel
- `minimalist` - Simple, uncluttered
- `dark` - Moody, sleek appearance
- `neon` - Vibrant, cyberpunk vibes
- `pastel` - Soft, muted colors
